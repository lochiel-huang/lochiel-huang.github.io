CASE 01

audit-compliance-copilotaudit-compliance-copilot（審計合規副駕）

Role: Solo build · Python + Streamlit + DeepSeek · Deployed (2026)角色：獨立開發 · Python + Streamlit + DeepSeek · 已上線（2026）

A two-layer AI system for accounting voucher anomaly detection. Layer one is a deterministic rule engine that catches structural violations (missing fields, amount mismatches, invalid account codes). Layer two is an LLM judgement layer that reads the surviving vouchers and flags contextual anomalies a rule engine can't see — the kind of thing a human auditor would notice on the second pass.

一個兩層架構的 AI 系統，用來偵測會計憑證的異常。第一層是確定性的規則引擎，抓結構性違規（欄位缺失、金額不符、無效科目代碼）。第二層是 LLM 判斷層，讀通過第一層的憑證，標記規則引擎看不到的上下文異常——那種人類審計師在第二遍會注意到的東西。

The three-iteration story三次迭代的故事

v1 flagged 77% of vouchers as suspicious. Alert fatigue — nobody would use it. v2 tightened the LLM prompt too hard, suppressed everything down to 7%, and started missing real anomalies. v3 rebalanced with explicit reasoning steps and calibrated thresholds; suspicious rate settled at ~30%, with the flagged items actually being worth reviewing. This arc — over-alerting → over-suppression → calibration — is exactly the evaluation-driven iteration mindset I want to work with.

v1 把 77% 的憑證標記為可疑。警報疲勞——沒人會用。v2 把 LLM prompt 收得太緊，全部壓到 7%，開始漏掉真的異常。v3 用明確的推理步驟和校準過的閾值重新平衡；可疑率穩定在約 30%，而且標記出來的都值得看。這個弧線——過度警報 → 過度抑制 → 校準——就是我想做的評估驅動迭代的思路。

Design choices worth explaining值得說明的設計選擇
  • DeepSeek over Anthropic. Anthropic's API requires payment methods unavailable in Hong Kong. DeepSeek's OpenAI-compatible API resolved this mid-project — a small logistics detail that mattered.
  • 用 DeepSeek 不用 Anthropic。Anthropic API 需要香港沒有的支付方式。DeepSeek 的 OpenAI 相容 API 中途解決了這個問題——一個小的物流細節，但很關鍵。
  • Ground truth is separate. The data generator produces ground truth files decoupled from evaluation, so the rule engine can't accidentally learn the answers.
  • ground truth 獨立產生。資料產生器將 ground truth 檔案與評估流程分離，避免規則引擎意外「學到」答案。

CASE 02

Field Observation Report: Social Interaction Patterns田野觀察報告：社交互動行為分析

Role: Observer & Analyst · Personal research · Documented (June 2026)角色：觀察者與分析者 · 個人研究 · 已記錄（2026年6月）

The setting. A late-night social venue. A stranger initiated contact and, over two hours, deployed a sequence of persuasion tactics: a purity narrative, appeals to feminist principles, criticism of materialistic customs, and a physically escalating request near departure. My interest was not in the individual encounter but in the structure: how consistently the tactics escalated, how the earlier statements retroactively looked like scaffolding for the later ones.

場景。一個深夜社交場所。一名陌生人主動搭訕，在兩小時內部署了一系列說服策略：純愛敘事、訴諸女權主義、批評物質化習俗，以及在離開前的肢體升級請求。我感興趣的不是這次個別相遇，而是它的結構：策略升級的一致性，以及前面的話語事後看起來像是為後來做的鋪墊。

Method方法

Following the encounter, I wrote up the sequence as a structured behavioural log: numbered event codes, timestamps, verbal and non-verbal moves, and a clear distinction between observed behaviour and observer inference.

相遇結束後，我把整個過程整理成結構化行為記錄：編號的事件代碼、時間戳、語言和非語言動作，以及對觀察行為和觀察者推斷的明確區分。

Why this counts as work為什麼這算是作品
  • Discomfort as data. The impulse to react emotionally is real; the value is in converting it into something structured that someone else could read and interrogate.
  • 不適感作為數據。情緒反應的衝動是真實的；價值在於把它轉化成別人可以閱讀和質疑的結構化內容。
  • Method transfer. The same taxonomy-and-annotation reflex is what I use in compliance work. The medium is different; the discipline is identical.
  • 方法遷移。這套分類和標注的反射動作，正是我在合規工作中使用的。媒介不同，紀律相同。
